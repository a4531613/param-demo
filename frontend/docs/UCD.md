# 参数配置中心（UCD 说明）

## 用户角色

- 管理员（admin）：拥有全量配置与发布权限，负责治理与审计。
- 配置维护（editor）：负责日常字段/数据维护与版本发布，受治理规则约束。
- 只读（viewer）：仅查看与导出，不允许新增/修改/删除/导入。
- 审计（auditor）：以审计为主，查看操作轨迹与关键数据，不允许写入。

## 核心使用场景

- 初始化：创建应用、环境、类型，建立“配置域”边界。
- 变更：创建版本、维护字段、录入数据，形成可追溯的版本化变更。
- 发布：将版本发布到可用态，归档历史版本，避免误操作。
- 运营：一键预览、导出 HTML、对比版本差异、查看审计日志。

## 主路径（建议顺序）

1. 应用 → 2. 环境 → 3. 配置类型 → 4. 版本 → 5. 字段 → 6. 配置数据 → 7. 预览/导出/对比 → 8. 审计闭环

## 交互与反馈规范（前端落地）

- 统一反馈：成功/警告/错误使用统一的 toast 与 confirm 文案与时长（`frontend/src/ui/feedback.js`）。
- 降低误操作：删除、批量删除、跨环境删除统一二次确认。
- 能力约束：只读/审计角色在 UI 层禁用写操作（按钮禁用 + 统一提示）。
- 空状态引导：列表为空时给出下一步动作，减少“无从下手”的认知负担。
